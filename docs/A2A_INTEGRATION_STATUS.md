# A2A 統合の進捗と修正内容まとめ (2026-04-03)

OpenCode と Gemini CLI を Agent-to-Agent (A2A) プロトコルで接続するための統合作業の成果をまとめます。

## 1. 完了した主な修正

### プラグイン側 (`opencode-geminicli-a2a`)
*   **Bun セグメンテーション違反の回避**: `src/index.ts` からロジックを `src/client.ts` に分離し、Bun v1.3.11 環境でのプラグイン読み込み時のクラッシュを修正しました。
*   **レスポンス形式の適合**: `usage` オブジェクトを AI SDK 仕様 (`{ inputTokens: 0, outputTokens: 0 }`) に合わせ、OpenCode UI でのエラーを解消しました。
*   **ストリーム終了判定の改善**: 受信テキストが空のままストリームが終了した場合の例外処理を追加しました。

### A2A サーバー側 (`gemini-cli` OSS)
*   **モデル選択の動的反映**: クライアントから送られるメタデータ (`message.metadata.coderAgent.model`) を解釈し、サーバー側で動的にモデル（Flash/Pro等）を切り替える機能を実装しました。
*   **無限ループの解消 (重要)**: 
    *   サーバー側の `executor.ts` にパッチを適用し、A2A バックエンドとして動作する際は **ツールの実行を強制的に無効化** (`config.getTools = () => []`) するようにしました。
    *   これにより、`GEMINI.md` 等の指示によってサーバー側が自動的に `activate_skill` を呼び出し、承認待ち (`input-required`) 状態でスタック・リトライを繰り返すループを根本から断ち切りました。
*   **ルーティング・バイパス**: `modelRouterService.ts` を修正し、サーバー内部のルーティング戦略をスキップして、指定されたモデルを直接使用するようにしました。

## 2. 現在のステータスと課題

*   **モデル制限 (Quota)**: `gemini-3-flash-preview` が利用上限に達したため、現在は `gemini-3.1-pro-preview` 等の別モデルでの検証フェーズに移行しています。
*   **OpenCode 設定**: `opencode.jsonc` にてローカルビルドのプラグイン (`file:///...`) を使用し、且つ `gemini-3.1-pro-preview` をホワイトリストに追加して動作確認中です。

## 3. 今後の予定
*   モデル制限解除後の Flash モデルでの最終確認。
*   A2A 通信におけるエラーハンドリングのさらなる強化。
